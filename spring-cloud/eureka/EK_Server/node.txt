用来当做eureka的服务端  
服务器自我保护机制
	当一个服务提供者出现问题，心跳测试失败率超过一定比例，或者超过十五分钟没有心跳测
	试，则服务器会把者服务看做是出现问题，并且不会马上从服务列表清楚这个服务。但是这
	样会出现客户端可以调用服务，但是会发现这个服务不能用。后面会有容错机制，也可以关
	闭自我保护
1、心跳测试
eureka:
  client:
    register-with-eureka: false
    fetch-registry: false    #注册到服务器，false为不注册
  server:
    enable-self-preservation: false   #服务器的自我保护，默认为true，   设置为false就是关闭
    eviction-interval-timer-in-ms: 5000  #每次清理列表的间隔时间，从没有心跳测试开始计时
